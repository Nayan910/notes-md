# Setup & Run Instructions

> All commands assume you're in `E:\oprncode\project\`.

## Quick Start (Development Mode)

Open three terminals:

```powershell
# Terminal 1: Backend API
cd backend\notes-md-api
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
# → http://localhost:8000/docs (OpenAPI)
# → http://localhost:8000/health

# Terminal 2: Web Editor
cd apps\notes-md
npm install
npm run dev
# → http://localhost:5173

# Terminal 3: Flutter (once SDK is set up)
cd apps\notes-md-app
$env:PATH = "E:\apps\flutter\flutter\bin;$env:PATH"
flutter pub get
flutter run
```

## Prerequisites

| Tool | Version | Install | Check |
|------|---------|---------|-------|
| Python | ≥3.10 | System install | `python --version` |
| Node.js | ≥18 | `winget install OpenJS.NodeJS` | `node --version` |
| Flutter | 3.38+ | Already at `E:\apps\flutter\flutter\bin` | `flutter --version` |
| npm | ≥10 | Ships with Node.js | `npm --version` |
| Android SDK | — | Run `setup-android.ps1` | `flutter doctor` |
| JDK | 17+ | https://adoptium.net | `java --version` |

## Setup Android SDK

```powershell
# Run the setup script (downloads CLI tools + installs platform 35)
.\setup-android.ps1

# Then configure Flutter
$env:ANDROID_HOME = "E:\apps\android-sdk"
$env:ANDROID_SDK_ROOT = "E:\apps\android-sdk"
$env:PATH = "E:\apps\android-sdk\platform-tools;$env:PATH"

# Accept licenses
flutter doctor --android-licenses
```

## Build for Production

```powershell
# Web editor (production build)
cd apps\notes-md
npm run build
# Output: apps/notes-md/dist/

# Flutter APK (Android)
cd apps\notes-md-app
flutter build apk --debug
# Output: build/app/outputs/flutter-apk/app-debug.apk

# Flutter Windows
flutter build windows
# Output: build\windows\runner\Release\
```

## Run Tests

```powershell
# Backend tests
cd backend\notes-md-api
python -m pytest test_api.py -v

# Web editor (when tests exist)
cd apps\notes-md
npm test

# Flutter tests
cd apps\notes-md-app
flutter test
```

## Important Config Values (NEED YOUR IP)

These files contain hardcoded IP addresses that need to be updated to your machine's local network IP:

| File | Line | Current Value | Replace With |
|------|------|--------------|--------------|
| `backend/notes-md-api/pairing.py` | 22 | `http://localhost:8000` | `http://YOUR.LOCAL.IP:8000` |
| `apps/notes-md-app/lib/screens/home_screen.dart` | 14 | `http://192.168.1.100:5173` | `http://YOUR.LOCAL.IP:5173` |

The QR code generated by `/pair/generate` includes the server URL. For Android to connect, it needs to reach your machine on the local network. Use `ipconfig` to find your IP.
