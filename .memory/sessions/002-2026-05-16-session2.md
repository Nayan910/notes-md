## Key Decisions Made

1. **Auth approach:** Server-based auth (bcrypt + JWT) + QR device pairing (WhatsApp Web model)
2. **WhatsApp Web pattern:** Web app shows QR, Android scans to claim
3. **QR data format:** JSON with type/version/token/server fields
4. **Polling vs WebSocket:** Polling (every 2s) for QR claim status — simpler, no WebSocket needed
5. **Provider over Riverpod:** Simpler for this use case, fewer concepts to learn
