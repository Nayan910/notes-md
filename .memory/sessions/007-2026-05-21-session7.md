## What was done

### Features implemented

1. **WebView error logging** — `onConsoleMessage` now filters errors/warnings to `AppLogger`. Added `onLoadError` and `onLoadHttpError` handlers to catch WebView load failures.

2. **In-app log viewer** — `AppLogger` service (500-entry circular buffer) + `LogViewer` widget (accessible via bug-icon button in toolbar → `endDrawer`). Entries have level icons (info/warn/error) and timestamps.

3. **.md file association (Android)** — AndroidManifest.xml now has `VIEW` intent filters for `.md` files (`text/markdown` + `text/plain` with `pathPattern=".*\\.md"`, both `file://` and `content://` schemes). Native `MainActivity.kt` reads the file from the intent and sends it to Flutter via `MethodChannel`. Dart `main.dart` stores it in `IncomingFile` singleton; `EditorScreen` reads it when the bridge is ready.
