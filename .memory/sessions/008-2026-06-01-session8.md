## What was done

### Architecture: Offline-first refactor

The app was refactored to work **fully offline with no server required**:

1. **ServerConfigService** (`services/server_config_service.dart`) ŌĆö manages optional server URL from `FlutterSecureStorage`. Default is `null` (fully offline). User can set a URL in settings to enable sync features.

2. **SettingsDialog** (`widgets/settings_dialog.dart`) ŌĆö dialog with:
   - Server URL text field (with clear button)
   - Connection status indicator (green cloud icon when configured, grey cloud-off when offline)
   - Sync toggle (only visible if server is configured)
   - About section with version info

3. **HomeScreen rewrite** (`screens/home_screen.dart`) ŌĆö completely replaced the old server-URL WebView with:
   - Loads web editor HTML from bundled Flutter assets (`rootBundle.loadString`)
   - Platform-aware base URL: `file:///android_asset/notes-md/` (Android), `file:///data/flutter_assets/assets/notes-md/` (Windows/Linux)
   - Bridge service with proper JavaScript handler and `flutter_postMessage` injection
   - Settings gear icon + bug report icon in toolbar
   - Sync status indicator (green cloud when connected)
   - Proper `onLoadError`/`onLoadHttpError`/`onConsoleMessage` handlers

4. **main.dart** ŌĆö app launches directly into `HomeScreen` (no login/loading screens). Uses `MultiProvider` for both `AuthService` + `ServerConfigService`.
